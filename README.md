# Аналитика рекламных кампаний в Apache Superset

## Описание проекта

Проект представляет собой интерактивный BI-дашборд для анализа эффективности рекламных кампаний, созданный в Apache Superset.

Дашборд позволяет отслеживать ключевые маркетинговые и продуктовые показатели, анализировать динамику рекламных кампаний, сравнивать каналы привлечения, типы рекламы и поведение пользователей на разных устройствах.

В рамках проекта был создан единый комбинированный датасет на основе трёх исходных датасетов. На его основе были рассчитаны ключевые метрики и визуализированы в виде KPI, графиков и аналитических вкладок.

## Цель проекта

Цель проекта — создать удобный инструмент для анализа рекламных кампаний, который помогает:

- оценивать эффективность маркетинговых вложений;
- отслеживать динамику выручки, затрат и ROI;
- сравнивать рекламные каналы и типы рекламных кампаний;
- анализировать пользовательское поведение по устройствам;
- быстро находить наиболее эффективные источники трафика.

## Подготовка данных

В рамках проекта был создан единый комбинированный датасет на основе трёх исходных датасетов.

На уровне датасета были рассчитаны основные метрики, которые затем использовались в визуализациях и KPI-индикаторах.

## Основные метрики

На дашборде представлены следующие ключевые показатели:

- **Выручка** — сумма доходов за выбранный период с сравнением с предыдущим периодом.
- **Затраты** — расходы на рекламу с динамикой по месяцам.
- **ROI** — возврат инвестиций, рассчитанный по формуле:  
  `(Выручка - Затраты) / Затраты`
- **Количество заказов** — число заказов за выбранный период.
- **Конверсия в заказ** — доля пользователей, совершивших хотя бы один заказ после перехода из рекламы.
- **Среднее время сессии** — средняя длительность пользовательской сессии.
- **Количество сессий** — количество уникальных сессий, рассчитанное по комбинации пользователя, даты захода, устройства и рекламного канала.

## Структура дашборда

Дашборд состоит из нескольких аналитических вкладок.

### 1. KPI / Основные метрики

В верхней части дашборда размещены ключевые показатели эффективности:

- выручка;
- затраты;
- ROI;
- количество заказов;
- конверсия в заказ;
- среднее время сессии.

KPI расположены в логической последовательности: от финансовых показателей к операционным и пользовательским метрикам. Для KPI реализовано сравнение с предыдущим периодом — PoP, period-over-period.

### 2. Тип рекламной кампании

Вкладка предназначена для анализа эффективности разных типов рекламных кампаний.

Визуализации:

- **ROI по типу рекламы** — линейный график, показывающий динамику возврата инвестиций по типам рекламных кампаний.
- **Баланс доходов и расходов по типу рекламы** — столбчатая диаграмма, где доходы отображаются как положительные значения, а расходы как отрицательные. Это помогает наглядно оценить прибыльность разных типов рекламы.

### 3. Рекламный канал

Вкладка помогает анализировать эффективность каналов привлечения пользователей.

Визуализации:

- **Динамика сессий по каналам привлечения** — столбчатая диаграмма, показывающая количество пользователей или сессий по рекламным каналам.
- **ROI по каналам привлечения** — линейный график для сравнения эффективности каналов.
- **Средняя длительность сессии по каналам** — график, показывающий, сколько времени пользователи проводят в приложении в зависимости от источника трафика.

### 4. Тип устройств

Вкладка показывает различия в поведении пользователей в зависимости от устройства.

Визуализации:

- **Количество сессий по девайсам, по месяцам** — столбчатая диаграмма для анализа активности пользователей на разных устройствах.
- **Распределение сессий по девайсам** — круговая диаграмма, показывающая долю устройств в общем количестве сессий.
- **Количество сессий по девайсам, по дням** — линейный график ежедневной динамики активности пользователей.

## Фильтры

На дашборде реализованы фильтры:

- **Период** — выбор временного диапазона данных;
- **Гранулярность** — настройка детализации, по умолчанию месяц;
- **Тип рекламы** — анализ отдельных типов рекламных кампаний;
- **Рекламный канал** — сравнение источников трафика;
- **Тип устройства** — анализ поведения пользователей на разных устройствах.
